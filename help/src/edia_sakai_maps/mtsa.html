Maps: Permissions

The Maps tool uses the following permissions:

site.visit
Not a permission defined by the tool, but a permission implied by Sakai. This permission is required to view the maps page of the tool. So if you want to provide user that are not logged in access to the tool, the .anon user needs the site.visit permission.
sakaimaps.createmarker
The permission to create markers.
sakaimaps.editmarker
The permission to edit all existing markers. Please note that people can always edit the markers they created themselves, even if they do not have this permission set.
sakaimaps.deletemarker
The permission to delete all existing markers. Please note that people can always delete the markers they themselves created, even if they do not have this permission set.
sakaimaps.admintypes
The permission to configure the different marker types of the map. See the section about managing types for details.
sakaimaps.admintool
The permission the configure the general tool settings, please see the section about tool administration for details.
